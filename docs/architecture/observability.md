# 可观测性标准

## 日志

- 统一 JSON 日志，必须字段：`trace_id`, `span_id`, `route`, `status`, `latency_ms`, `error_code`。
- 业务错误与系统错误分级记录，禁止记录敏感明文信息。

## 指标

- 流量：QPS、并发连接数。
- 性能：P50/P95/P99 延迟。
- 稳定性：错误率、超时率、重试率。
- 资源：CPU、内存、连接池水位、队列积压。

## Trace

- 所有入站请求生成或透传 `x-trace-id`。
- 出站调用必须透传 trace 上下文，确保跨服务链路关联。

## SLO 建议

- 核心写接口可用性 >= 99.9%。
- 核心写接口 P99 < 800ms（稳态）。
- 错误率 < 0.3%（5 分钟窗口）。
