# 用户入口高变入口配置化原则记录（2026-04-09）

## 背景

在 `用户统一入口`、`用户端细化块规划`、`交易主链路块规划` 已基本成形后，发现一个持续风险：

- 如果首页活动位、增长入口、导购入口继续直接写死在前台结构中
- 后续运营扩张仍可能重新干扰交易主链路
- 文档虽然能提示“不要膨胀”，但缺少更稳定的结构约束

## 本次决策

统一把用户入口中的高变入口定义为：

- 前台承接稳定入口容器
- 平台后台负责入口配置

统一原则为：

- 主链路结构固定
- 高变入口配置化

配置化范围主要包括：

- 首页活动位
- 首页增长位
- 首页导购位
- 会员权益/积分商城首页快捷入口
- 推广中心首页露出
- 客服浮层策略
- 登录后强引导弹窗和待办弹窗策略

## 原因

- 用户统一入口首先要保护交易主链路，而不是优先满足短期运营编排
- 高变入口天然会频繁变化，不适合长期写死在 IA 中
- 如果每次活动、增长或导购调整都改入口结构，后续文档会反复失稳
- 采用“固定坑位 + 后台配置”后，可以把变化收敛到配置层，而不是结构层

## 影响范围

- `meeting/5-1用户统一入口.md` 应只冻结稳定 IA，不写死具体活动内容
- `meeting/5-1-2用户端细化块规划.md` 中的高变入口都应按配置化原则理解
- `meeting/5-1-2-1交易主链路块规划.md` 中首页承接只负责稳定容器与优先级，不负责具体运营内容
- `meeting/5-50-用户入口结构图解.md` 中所有活动、增长、导购露出位都应理解为前台容器，而不是固定活动本体

## 风险

- 如果后台配置能力边界不清，仍可能借“配置化”名义突破主链路优先级
- 如果没有默认态和失效回退机制，配置失误仍可能伤害首页可用性
- 如果活动内容、跳转目标、人群与时段不受约束，会导致入口语义漂移

## 约束

- 配置化入口不得改写底部一级导航
- 配置化入口不得改写交易主链路层级
- 配置化入口不得破坏支付结果后的回落路径
- 配置化入口只能在既定容器内调整显隐、排序、跳转、人群和时段
- 配置失效时必须自动回退到稳定默认态

## 后续建议

- 后续继续细化首页承接时，应单独补一份“入口配置后台约束”文档
- 如后续新增活动会场、增长玩法、导购露出位，应优先回答“挂在哪个固定容器里”，而不是先改 IA
- 后续如果配置项明显增多，可再补“入口配置字段清单”和“优先级冲突处理规则”
