# 用户入口购物车结算栏约束记录（2026-04-09）

## 背景

在查看 `meeting/5-50-用户入口结构图解.md` 的购物车页结构时，明确需要把底部结算栏从“示意说明”升级为“页面级固定约束”，避免后续被实现成普通列表尾部模块。

## 本次结论

- 购物车页中的底部结算栏在用户停留该页时应常驻显示
- 商品列表区域负责滚动，结算栏不跟随列表滚动离开视口
- 结算栏固定在底部 Tab 上方，作为购物车页的核心操作区
- 页面内容区需为结算栏预留安全高度，避免最后一项商品被遮挡

## 影响范围

- 后续购物车页原型、前端布局与交互说明应默认遵守该约束
- 若后续引入失效商品区展开、推荐商品区或营销提示，也不得挤占底部结算栏的常驻位置
