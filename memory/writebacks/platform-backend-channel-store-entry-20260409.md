# 平台后台入口补充：渠道店铺治理口径（2026-04-09）

## 背景

在讨论 `meeting/5-2平台后台入口.md` 时，确认 `渠道店铺` 虽然已经作为平台后台一级菜单存在，但原文对第三方渠道接入、店铺治理、与渠道同步中心的边界描述仍偏弱。

## 本次统一口径

- `渠道店铺` 对应挂载业务端域中的 `外部渠道与店铺管理端`，不是普通系统配置子页面。
- 该模块负责治理与配置，承接第三方平台接入、店铺授权、能力矩阵、映射管理、店铺状态治理。
- `渠道同步中心` 负责商品、订单、库存、客服消息等同步执行，不替代 `渠道店铺` 的治理职责。
- 二者关系统一为“治理台提供配置与策略，同步执行层按配置运行任务”。
- 多平台接入需支持同平台多店并行管理，授权、限流、故障隔离按店铺粒度处理。
- 店铺需支持绑定品牌、供应商、仓库、价格策略、物流模板等内部对象。
- 渠道能力差异必须通过能力矩阵显式标注，高风险动作需纳入权限、审批与审计。

## 影响范围

- `meeting/5-2平台后台入口.md` 中 `渠道店铺` 的定位更明确，与 `meeting/20260407.md` 和 `meeting/6端域边界.md` 的口径对齐。
- 后续若继续细化菜单、权限或一期范围，应以“治理层”和“同步执行层”分离为前提，不再把该模块表述为单纯配置页。
