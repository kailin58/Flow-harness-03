# 用户入口结算页提交订单栏约束记录（2026-04-09）

## 背景

在明确购物车页底部结算栏“页面内常驻显示”后，交易主链路下一步的结算页也需要统一口径，避免购物车页与结算页在核心操作区设计上出现不一致。

## 本次结论

- 结算页中的底部提交订单栏在用户停留该页时应常驻显示
- 结算内容区负责滚动，提交订单栏不跟随内容区滚动离开视口
- 提交订单栏稳定承载应付金额与 `提交订单` 主动作，作为结算页核心操作区
- 结算内容区需为底部提交订单栏预留安全高度，避免最后一块确认信息被遮挡

## 影响范围

- 后续结算页原型、前端布局与提单交互说明应默认遵守该约束
- 若后续增加地址卡、优惠区、积分抵扣、风险提示等模块，也不得挤占底部提交订单栏的常驻位置
