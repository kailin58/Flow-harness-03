# 用户端交易主链路固定操作区规范记录（2026-04-09）

## 背景

在分别明确商品详情页、购物车页、结算页的底部常驻主操作区后，相关结论已经分散在结构图解、主链路规划和若干 writeback 中。为避免后续继续逐页重复解释，需要抽出一份统一规范文档。

## 本次产出

- 新增 `meeting/5-1-2-1-0交易主链路固定操作区规范.md`

## 规范覆盖范围

- 商品详情页：`加入购物车 / 立即购买`
- 购物车页：`合计金额 / 去结算`
- 结算页：`应付金额 / 提交订单`

## 统一结论

- 三类页面统一采用“页面内常驻显示”的底部固定操作区
- 主体内容区负责滚动，固定操作区不跟随滚动离开视口
- 页面内容区必须为固定操作区预留安全高度
- 固定操作区只承载当前页最关键的成交动作

## 影响范围

- 后续原型、视觉、前端实现可直接引用该规范，不再逐页重复争论
