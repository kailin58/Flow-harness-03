# 用户入口结构图解 — H5 端布局与子页面关系

## 文档定位

基于 `5-1` 用户统一入口 IA、`5-1-1` 增强模块定位、`5-1-2` 细化块规划及全部下钻文档，输出 H5 端的**页面结构布局**与**子页面间导航关系**。

本文件不重复业务口径，仅以图解形式呈现"用户在 H5 端看到什么、能去哪里"。

---

## 1. H5 全局框架

```
┌─────────────────────────────┐
│城市│ 💬 问我任何事...        │  ← 统一 Chat 入口栏（P0）
├─────────────────────────────┤
│                             │
│      当前 Tab 页面内容       │  ← 主体内容区（可滚动；稳定容器）
│                             │
│                             │
├─────────────────────────────┤
│  首页  分类 🔔  购物车  订单  我的 │  ← 底部一级导航（P0 冻结）
└─────────────────────────────┘

全局浮层：
  [客服浮层按钮]  ← 右下角固定，不占内容区面积
```

### 全局元素

| 元素 | 位置 | 说明 |
|---|---|---|
| 统一 Chat 入口栏 | 首页顶部常驻 | 城市 + Chat 输入框；搜索/对话两种能力统一收口到 chat 交互，详见第 2.1 节 |
| 主体内容区 | Chat 入口栏与底部一级导航之间 | 当前 Tab 的稳定承载容器；当当前 Tab 为首页时，首页展示内容按固定坑位由平台后台配置 |
| 消息入口 🔔 | 底部导航栏「分类」旁 | 消息通知唯一定位入口，登录态显示未读角标 |
| 底部一级导航 | 底部固定 | 首页 / 分类 🔔 / 购物车 / 订单 / 我的，不可配置改动 |
| 客服浮层 | 右下角悬浮 | 全局可见，可手动收起；收起后在「我的」仍可找到 |

**主体内容区说明**：

- 主体内容区是当前 Tab 的稳定承载容器，负责承接首页、分类、购物车、订单、我的各自页面内容
- 该容器本身属于前台结构层，不因活动、增长或导购配置而改变层级
- 当当前 Tab 为首页时，首页内频道区、商品区、活动位和增强露出位的具体展示内容由平台后台配置
- 平台后台配置的是容器内的内容、显隐和排序，不是改写 H5 全局框架

## 2. 五个一级 Tab 页面布局

### 2.1 首页

```
┌─────────────────────────────┐
│城市│ 💬 问我任何事...        │  ← 统一 Chat 入口栏（P0 常驻）
│    │  搜索/对话统一交互入口   │     城市 + Chat输入框
├─────────────────────────────┤

│  │  商品卡  │ │  商品卡  │   │     后台配置选品与排序
│  │  图+标题  │ │  图+标题  │   │     坑位结构冻结
│  │  ¥价格   │ │  ¥价格   │   │     点击→商品详情
│  └─────────┘ └─────────┘   │
│  ┌─────────┐ ┌─────────┐   │
│  │  商品卡  │ │  商品卡  │   │
│  │  ...    │ │  ...    │   │
│  └─────────┘ └─────────┘   │
├─────────────────────────────┤
│  ┌───────────────────────┐  │  ← 运营与增长露出区（P2-P4，全部配置化）
│  │ 活动坑位               │  │     活动坑位（P2）
│  │ 衣橱/行程规划露出位    │  │     衣橱/行程规划（P3，默认不露出）
│  │ 推广/积分/会员露出位   │  │     推广/积分/会员（P4，默认不露出）
│  └───────────────────────┘  │
├─────────────────────────────┤
│  首页  分类 🔔  购物车  订单  我的 │
└─────────────────────────────┘
                     💬 ← 客服浮层
```

**统一 Chat 入口栏说明**：

顶部 Chat 输入框是搜索、对话两种能力的**统一交互入口**，用户在任何意图下都通过同一个 chat 交互完成：

| 用户输入 | Chat 交互行为 | 结果呈现 |
|---|---|---|
| "黑色连衣裙" | 识别为搜索意图 | 返回商品结果卡（可点击→商品详情） |
| "帮我搭一套通勤装" | 识别为穿搭顾问意图 | 进入对话模式，推荐搭配卡片 |
| "我的订单到哪了" | 识别为查询意图 | 返回订单/物流状态卡 |

**注意**：消息通知不在 Chat 入口栏内，统一由底部导航栏「分类 🔔」进入。

**搜索下方模块快捷栏说明**：

- 位于统一 Chat 入口栏下方，是首页内的稳定模块容器
- 当前首批先占位 `天气`、`商城`、`衣橱`、`行程规划`
- `商城` 点击后按用户身份分流：会员进入客户端商城，分销进入自己商城的首页
- 后续可继续新增模块，不预设固定上限
- 前端展示形态写死为单栏横向滑动，不允许因模块增多改成纵向堆叠并持续挤压首页主内容区
- 平台后台只配置该栏中的模块内容、顺序、显隐、身份命中规则与跳转，不改写该栏展示形态与首页整体结构

**设计约束**：
- Chat 入口栏为 P0 常驻，不可配置隐藏或移位
- 搜索下方模块快捷栏位置固定在 Chat 下方，不新增底部 Tab
- 输入框默认 placeholder 为"问我任何事…"，可通过后台配置文案
- Chat 交互内返回的商品卡可直接加购/跳转详情，不中断对话
- 对话历史可在「我的 → 对话记录」中查看

**首页优先级冻结**：P0 统一Chat入口 > P1 搜索下方模块快捷栏/商品/分类 > P2 活动 > P3 增强入口/客服 > P4 增长露出

购物车页补充约束：

- 底部结算栏在购物车页内常驻显示，只要用户停留在该页就不隐藏
- 商品列表区域负责滚动，结算栏不随列表滚动离开视口
- 结算栏固定在底部 Tab 上方，作为购物车页的核心操作区
- 页面内容区需为底部结算栏预留安全高度，避免最后一项商品被遮挡

### 3.2 交易主链路子页面一览

| 子页面 | 入口来源 | 可跳转到 |
|---|---|---|
| 搜索结果页 | 首页搜索、分类页搜索 | 商品详情 |
| 分类列表页 | 首页分类入口、底部 Tab「分类」 | 商品详情、搜索结果页 |
| 商品详情页 | 搜索结果、分类列表、首页商品卡、活动坑位、购物车 | 购物车、结算、客服、对话入口 |
| 购物车页 | 底部 Tab「购物车」、商品详情加购后 | 商品详情（补货/回看）、结算 |
| 结算页 | 购物车去结算、商品详情立即买 | 支付页 |
| 支付页 | 结算页提交订单 | 支付结果页 |
| 支付结果页 | 支付完成后自动跳转 | 首页、订单详情 |

商品详情页补充约束：

- 底部主操作区在商品详情页内常驻显示，只要用户停留该页就不隐藏
- 商品详情内容区负责滚动，底部主操作区不跟随内容区滚动离开视口
- 底部主操作区稳定承载 `加入购物车` 与 `立即购买` 两个主动作
- 商品详情内容区需为底部主操作区预留安全高度，避免最后一段决策信息被遮挡

结算页补充约束：

- 底部提交订单栏在结算页内常驻显示，只要用户停留该页就不隐藏
- 结算内容区负责滚动，提交订单栏不跟随内容区滚动离开视口
- 提交订单栏作为结算页核心操作区，应稳定承载应付金额与 `提交订单`
- 结算内容区需为底部提交订单栏预留安全高度，避免最后一块确认信息被遮挡

## 7. 关键路径流程

### 7.1 交易主链路完整路径

```
Chat入口 ──搜索意图──> 商品结果卡 ──点击──> 商品详情 ──加购──> 购物车 ──结算──> 结算页
                                                                        │
首页/分类 ──分类点选──> 列表页 ──点击──> 商品详情 ──立即买──> 结算页 <─────┘
                                                         │
                                                   提交订单
                                                         │
                                                         ▼
                                                     支付页 ──> 支付结果页
                                                                   │
                                                           ┌───────┼───────┐
                                                           ▼       ▼       ▼
                                                        回首页  查看订单  再逛逛
```

### 7.2 售后发起路径

```
订单Tab ──> 订单列表 ──> 订单详情 ──> 售后申请页 ──提交──> 售后进度页
                              │                                      │
                              │                              驳回/待补材料
                              │                                      │
                              │                              ┌───────┤
                              │                              ▼       ▼
                              │                         补充材料  联系客服
                              │
                              └── 物流查询 ──> 售后申请页 ──> ...
```

### 7.3 统一 Chat 交互分发路径

```
统一 Chat 入口 ──> Chat 对话页 ──> AI 识别意图 ──> 智能分发
                                                  │
                        ┌──────────┬──────────┬───┴──────────┐
                        ▼          ▼          ▼              ▼
                   搜索意图    对话意图    查询意图        服务意图
                        │          │          │              │
                        ▼          ▼          ▼              ▼
                   商品结果卡  穿搭顾问    订单/物流卡     客服/售后
                        │     推荐搭配     退款进度卡
                        │          │          │
                        ▼          ▼          │
                   商品详情    衣橱/行程     │
                        │      (反向带动)    │
                        │          │          │
                        ▼          ▼          ▼
                   交易主链路  资产沉淀/场景  服务分流
                                          编排
                                           │
                                           ▼
                                    反向带动交易主链路
                                    (推荐商品→商品详情→购买)
```

### 7.4 游客 → 登录 → 合并路径

```
游客浏览首页/商品 ──加购──> 购物车(游客态) ──触发登录──> 登录页
                                                          │
                                                    登录成功
                                                          │
                                                          ▼
                                                    合并购物车上下文
                                                    合并来源归因参数
                                                          │
                                                          ▼
                                                     首页(登录态)
```

### 7.5 外部渠道导流路径

```
淘宝/抖音/小红书/... ──> 导流落地页(用户统一入口)
                              │
                        参数透传 + 归因记录
                              │
                              ▼
                    首页 / 商品详情 / 活动会场
                              │
                        进入交易主链路
```

## 9. 登录态与显隐规则

### 9.1 游客态可见

- 首页（浏览）
- 分类/搜索（浏览）
- 商品详情（浏览）
- 购物车（可加购，触发登录时跳登录页）

### 9.2 登录态额外可见

- 订单列表/详情/物流/售后（全部）
- 我的页全部入口
- 消息中心
- 对话模块
- 搜索下方模块快捷栏
- 衣橱/行程规划
- 会员权益/积分商城
- 结算/支付

### 9.3 推广员资格态

| 资格状态 | 推广中心显隐 | 说明 |
|---|---|---|
| 未申请 | 我的页入口置灰或隐藏 | 需先完成学习→测验→签协议→提交审核 |
| 审核中 | 我的页入口可见，显示审核中 | |
| 已通过 | 我的页入口可见，全功能开放 | 可配置首页露出 |
| 已冻结/已注销 | 我的页入口可见，显示冻结状态 | |

补充说明：

- 搜索下方模块快捷栏中的 `商城` 入口按用户身份命中不同落点
- 会员点击 `商城` 后进入客户端商城
- 分销用户点击 `商城` 后进入自己商城的首页

## 10. 配置化口径汇总

### 冻结结构（不可配置改动）

- 统一 Chat 入口栏常驻位置（顶部，城市 + Chat 输入框）
- 当前 Tab 主体内容区作为 H5 页面稳定容器的层级位置
- 首页搜索下方模块快捷栏的容器位置
- 消息入口唯一定位在底部导航栏「分类 🔔」
- 底部一级导航 5 个 Tab
- 首页频道与分类入口区域位置和数量上限
- 商品内容区为首页核心面积
- 订单列表状态分组
- 我的页核心入口区域

### 可配置项

| 配置项 | 配置方式 | 默认值 |
|---|---|---|
| 首页搜索下方模块快捷栏内容、顺序与身份命中规则 | 平台后台配置 | 天气 / 商城 / 衣橱 / 行程规划 |
| 首页商品选品与排序 | 平台后台配置 | 系统默认排序 |
| 活动坑位显隐/位置/内容/跳转 | 平台后台配置 | 不展示 |
| 对话入口首页露出开关与位置 | 平台后台配置 | 露出 |
| 衣橱/行程规划首页露出开关 | 平台后台配置 | 不露出 |
| 推广中心首页露出开关 | 平台后台配置 | 不露出 |
| 积分商城/会员权益首页快捷入口 | 平台后台配置 | 不露出 |
| 客服浮层展示策略 | 平台后台配置 | 露出 |
| 消息中心待办弹窗策略 | 平台后台配置 | 按规则触发 |
| 登录后强引导弹窗 | 平台后台配置 | 不触发 |

### 配置化统一约束

- 配置化入口不得突破优先级表的视觉权重限制
- 配置化入口不得改写底部一级导航
- 配置失效时自动回退到稳定默认态
- 不允许空白位或异常展示
