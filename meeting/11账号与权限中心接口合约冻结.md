# 11 账号与权限中心接口合约冻结

## 目的

在 `账号体系与权限规划`、`账号与权限中心菜单规划`、`账号与权限中心对外接口草案` 三份文档已确定接口分类与入参口径后，本文件冻结：

- 具体 HTTP 路径
- 协议风格（REST / 事件）
- Request / Response Schema（含字段类型与长度）
- 错误码表
- 生命周期事件 Kafka Topic 与 Payload

本文件是一期 Wave 1 中 E4（权限与审计底座）的接口契约真源，开发以此为依据。

## 总体约定

### 协议风格

| 类型 | 协议 | 说明 |
|------|------|------|
| 同步业务 API | REST + JSON | 面向 BFF 与内部模块 |
| 事件接口 | Kafka（Outbox） | 生命周期事件，异步解耦 |

### 通用规范

- 基础路径：`/api/v1/iam`
- 认证方式：Bearer Token（Keycloak OIDC）
- 字符编码：UTF-8
- 时间格式：ISO 8601（`2026-04-09T14:30:00+08:00`）
- 分页参数：`page`（从 1 开始）、`size`（默认 20，最大 100）
- ID 格式：UUID v7（时间排序）
- 幂等：写操作要求客户端传 `Idempotency-Key` 头

### 通用响应包装

```json
{
  "code": "IAM_200_0001",
  "message": "success",
  "data": { ... },
  "trace_id": "0af7651916cd43dd8448eb211c80319c"
}
```

- `code`：业务码，格式 `{系统}_{HTTP状态}_{业务序号}`
- `message`：可展示文案
- `data`：业务负载，出错时为 null
- `trace_id`：全链路追踪 ID

### 通用错误码前缀

| 前缀 | 含义 |
|------|------|
| `IAM_400_xxxx` | 客户端参数错误 |
| `IAM_401_xxxx` | 未认证 |
| `IAM_403_xxxx` | 无权限 |
| `IAM_404_xxxx` | 资源不存在 |
| `IAM_409_xxxx` | 冲突（重复注册等） |
| `IAM_422_xxxx` | 业务校验失败 |
| `IAM_500_xxxx` | 服务端内部错误 |

## 一期必须冻结的 6 类接口

### 1. 模块注册接口

用途：新模块接入统一权限模型，注册资源与动作清单。

#### POST `/api/v1/iam/modules`

**Request**

| 字段 | 类型 | 必填 | 长度/范围 | 说明 |
|------|------|------|-----------|------|
| module_code | string | Y | 3~64，小写字母+下划线 | 模块唯一编码，如 `risk_center` |
| module_name | string | Y | 2~128 | 模块名称 |
| module_type | enum | Y | - | `admin_module` / `supplier_module` / `internal_workbench` |
| entry_scope | enum | Y | - | `platform_admin` / `supplier_portal` / `cs_workbench` / `fulfillment` / `user_app` |
| owner_domain | string | Y | 3~64 | 归属业务域编码，如 `trade`、`aftersale` |
| identity_types | string[] | Y | 1~4 项 | `staff` / `supplier_user` / `member` / `system` |
| resources | ResourceDef[] | Y | 1~200 项 | 资源清单 |
| is_high_risk_module | boolean | Y | - | 是否高风险模块 |
| default_role_templates | RoleTemplateDef[] | N | 0~50 项 | 默认角色模板 |
| scope_dimensions | string[] | N | 0~10 项 | 数据范围维度，如 `supplier`、`brand`、`store`、`channel` |
| field_permissions | FieldPermDef[] | N | 0~50 项 | 字段权限定义 |
| approval_actions | string[] | N | 0~100 项 | 需审批的动作清单 |

**ResourceDef**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| resource_code | string | Y | 资源编码，如 `order` |
| resource_name | string | Y | 资源名称 |
| actions | string[] | Y | 可执行动作，如 `view`、`create`、`export`、`approve` |

**RoleTemplateDef**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| role_code | string | Y | 角色编码 |
| role_name | string | Y | 角色名称 |
| permissions | string[] | Y | 格式 `{resource}:{action}`，如 `order:view` |

**FieldPermDef**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| field_code | string | Y | 字段编码，如 `cost_price` |
| field_name | string | Y | 字段名称 |
| sensitivity_level | enum | Y | `low` / `medium` / `high` |

**Response** `201 Created`

```json
{
  "code": "IAM_201_0001",
  "message": "module registered",
  "data": {
    "module_id": "0191a2b3-c4d5-7e6f-8a9b-0c1d2e3f4a5b",
    "module_code": "risk_center",
    "resource_version": 1,
    "default_role_template_ids": [
      "0191a2b3-c4d5-7e6f-8a9b-0c1d2e3f4a6c"
    ],
    "registration_status": "active",
    "created_at": "2026-04-09T14:30:00+08:00"
  }
}
```

**错误码**

| 错误码 | HTTP | 说明 |
|--------|------|------|
| IAM_409_0001 | 409 | module_code 已注册 |
| IAM_400_0001 | 400 | 资源或动作格式不合法 |
| IAM_422_0001 | 422 | identity_types 含不支持的类型 |

#### GET `/api/v1/iam/modules/{module_code}`

查询单个模块注册信息。返回同上 `data` 结构。

#### GET `/api/v1/iam/modules`

分页查询已注册模块列表。

| Query 参数 | 类型 | 必填 | 说明 |
|-------------|------|------|------|
| entry_scope | string | N | 按入口过滤 |
| module_type | string | N | 按类型过滤 |
| page | int | N | 默认 1 |
| size | int | N | 默认 20 |

#### POST `/api/v1/iam/principals:resolve

用途：通过登录 Token 或业务 ID 反查统一主体。供 BFF / 内部模块调用。

| 字段 | 类型 | 必填 | 说明 |

| subject_id | string | Y | 对应类型的 ID 值 |

**Response** `200 OK`

返回同 `GET /principals/{principal_id}` 的 `data` 结构。

### 3. 统一鉴权接口

用途：给业务模块输出统一的权限判断结果。后端必须二次校验，不信任前端按钮显隐。

#### POST `/api/v1/iam/authorizations:check`

**Request**

| 字段 | 类型 | 必填 | 长度/范围 | 说明 |
|------|------|------|-----------|------|
| principal_id | UUID | Y | - | 统一主体 ID |
| module_code | string | Y | 3~64 | 模块编码 |
| resource | string | Y | 3~128 | 资源编码 |
| action | string | Y | 2~64 | 动作编码 |
| target_context | object | N | - | 目标上下文，如 `{"supplier_id": "xxx", "order_id": "xxx"}` |
| field_list | string[] | N | 0~100 | 需要校验的字段列表 |

**Response** `200 OK`

```json
{
  "code": "IAM_200_0003",
  "message": "success",
  "data": {
    "allow": true,
    "deny_reason": null,
    "matched_roles": ["order_admin"],
    "data_scope": {
      "org_scope": ["0191a2b3-xxxx-xxxx-xxxx-xxxxxxxxxxxx"],
      "supplier_scope": [],
      "brand_scope": ["*"],
      "store_scope": ["store_001", "store_002"],
      "channel_scope": [],
      "department_scope": []
    },
    "field_scope": {
      "visible_fields": ["order_id", "amount", "status"],
      "masked_fields": ["customer_phone"],
      "hidden_fields": ["cost_price", "settlement_amount"],
      "exportable_fields": ["order_id", "amount", "status"]
    },
    "need_approval": false,
    "need_second_verify": false
  }
}
```

**deny_reason 枚举**

| 值 | 说明 |
|----|------|
| no_role | 无匹配角色 |
| no_resource_permission | 无资源权限 |
| no_data_scope | 无数据范围 |
| no_field_permission | 无字段权限 |
| account_frozen | 账号已冻结 |
| module_not_registered | 模块未注册 |

**错误码**

| 错误码 | HTTP | 说明 |
|--------|------|------|
| IAM_401_0001 | 401 | 未认证 |
| IAM_404_0002 | 404 | module_code 不存在 |

#### POST `/api/v1/iam/authorizations:batchCheck`

批量鉴权，Request 为 `checks[]` 数组，Response 为 `results[]` 数组，结构与单次一致。上限 50 项。

### 5. 统一审计写入接口

用途：各业务模块统一写入审计事件，落库到 OpenSearch（经 Outbox + Kafka）。

#### POST `/api/v1/iam/audit-events`

**Request**

| 字段 | 类型 | 必填 | 长度/范围 | 说明 |
|------|------|------|-----------|------|
| event_type | enum | Y | - | 见下方枚举 |
| module_code | string | Y | 3~64 | 来源模块 |
| resource | string | Y | 3~128 | 操作资源 |
| action | string | Y | 2~64 | 操作动作 |
| operator_id | UUID | Y | - | 操作人 principal_id |
| target_id | string | Y | 1~128 | 操作目标 ID |
| target_type | string | N | 3~64 | 操作目标类型 |
| result | enum | Y | - | `success` / `failure` / `denied` |
| before_snapshot | object | N | 最大 10KB | 变更前快照 |
| after_snapshot | object | N | 最大 10KB | 变更后快照 |
| trace_id | string | Y | 32 | 链路追踪 ID |
| ip | string | N | 7~45 | 操作 IP（IPv4/IPv6） |
| device_id | string | N | 1~128 | 设备标识 |
| user_agent | string | N | 最大 512 | 客户端 UA |
| remark | string | N | 最大 1024 | 补充说明 |

**event_type 枚举**

| 值 | 说明 |
|----|------|
| login | 登录 |
| logout | 登出 |
| export | 导出 |
| config_change | 配置变更 |
| batch_operation | 批量操作 |
| approval | 审批动作 |
| high_risk_action | 高风险动作 |
| data_access | 数据访问（敏感数据查看） |
| permission_change | 权限变更 |

**Response** `202 Accepted`

```json
{
  "code": "IAM_202_0001",
  "message": "audit event accepted",
  "data": {
    "event_id": "0191a2b3-c4d5-7e6f-8a9b-0c1d2e3f4a5b",
    "accepted_at": "2026-04-09T14:30:00+08:00"
  }
}
```

说明：

- 本接口返回 202 表示已接受，异步落库到 OpenSearch
- 写入模块通过 Outbox 模式保证：业务事务 + 审计事件在同一事务内写入 outbox 表，再由 CDC 投递 Kafka → OpenSearch
- 调用方不需要直接发 Kafka 消息

**错误码**

| 错误码 | HTTP | 说明 |
|--------|------|------|
| IAM_400_0002 | 400 | event_type 不合法 |
| IAM_400_0003 | 400 | snapshot 超过 10KB |
| IAM_404_0002 | 404 | module_code 未注册 |

#### GET `/api/v1/iam/audit-events`

分页查询审计日志（从 OpenSearch 读取）。

### 6. 生命周期事件接口

用途：账号状态变化时，通过 Kafka 事件通知下游模块，实现权限收敛与入口能力自动调整。

#### 协议：Kafka Topic

- Topic 前缀：`iam.lifecycle.`
- 分区键：`principal_id`
- 投递保证：At Least Once（消费方需幂等）
- 序列化：JSON Schema

#### Topic 与事件映射

| Topic | 事件名 | 说明 |
|-------|--------|------|
| iam.lifecycle.staff | account.staff.onboarded | 员工入职 |
| iam.lifecycle.staff | account.staff.offboarded | 员工离职 |
| iam.lifecycle.staff | account.staff.transferred | 员工转岗 |
| iam.lifecycle.staff | account.staff.frozen | 员工冻结 |
| iam.lifecycle.staff | account.staff.unfrozen | 员工解冻 |
| iam.lifecycle.supplier | account.supplier.main.enabled | 供应商主账号启用 |
| iam.lifecycle.supplier | account.supplier.main.disabled | 供应商主账号停用 |
| iam.lifecycle.supplier | account.supplier.sub.changed | 供应商子账号变更 |
| iam.lifecycle.role | account.role.changed | 角色变更 |
| iam.lifecycle.scope | account.scope.changed | 数据范围变更 |

#### 事件 Payload 通用结构

```json
{
  "event_id": "0191a2b3-c4d5-7e6f-8a9b-0c1d2e3f4a5b",
  "event_type": "account.staff.frozen",
  "occurred_at": "2026-04-09T14:30:00+08:00",
  "principal_id": "0191a2b3-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "principal_type": "staff",
  "operator_id": "0191a2b3-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "trace_id": "0af7651916cd43dd8448eb211c80319c",
  "version": 1,
  "payload": {
    "changed_fields": ["account_status"],
    "before": { "account_status": "active" },
    "after": { "account_status": "frozen" }
  }
}
```

**各事件特有 payload 字段**

| 事件名 | payload 额外字段 |
|--------|-----------------|
| account.staff.onboarded | `org_id`, `position_id`, `initial_role_ids` |
| account.staff.offboarded | `offboard_reason`, `effective_at` |
| account.staff.transferred | `from_org_id`, `to_org_id`, `from_position_id`, `to_position_id`, `added_role_ids`, `removed_role_ids` |
| account.staff.frozen | `frozen_reason` |
| account.staff.unfrozen | `unfrozen_reason` |
| account.supplier.main.enabled | `supplier_id`, `supplier_name` |
| account.supplier.main.disabled | `supplier_id`, `disable_reason` |
| account.supplier.sub.changed | `supplier_id`, `sub_account_id`, `change_type`（`created`/`disabled`/`role_changed`） |
| account.role.changed | `added_roles`, `removed_roles` |
| account.scope.changed | `scope_type`, `added_ids`, `removed_ids` |

#### 消费方接入约束

- 消费方必须按 `principal_id` 做幂等
- 消费方应按 `version` 做版本兼容
- 收到 `offboarded` / `frozen` / `disabled` 事件时，消费方必须在 5 分钟内收敛权限缓存
- 消费方不允许跳过事件自行缓存权限结果超过 30 分钟

## 模块依赖关系

```
iam-auth（认证：Keycloak 集成、登录、Token 校验）
  └── iam-permission（授权：Casbin/OpenFGA、角色、数据范围、字段范围、审计）
       ├── 消费方：各业务模块（trade, catalog, aftersale, ...）
       └── 消费方：各 BFF（bff-admin, bff-supplier, bff-cs, bff-fulfillment）
```

- `iam-auth` 负责：登录、OIDC Token 签发与校验、统一身份解析
- `iam-permission` 负责：模块注册、鉴权决策、数据范围、字段范围、审计写入、生命周期事件

---

## 新模块接入检查清单

新模块接入权限体系时，按以下顺序检查：

- [ ] 1. 调用 `POST /modules` 注册 module_code、资源、动作
- [ ] 2. 在 `账号与权限中心` 治理台配置角色、数据范围、字段范围
- [ ] 3. 后端所有需鉴权入口调用 `POST /authorizations:check` 做二次校验
- [ ] 4. 所有高风险操作写入 `POST /audit-events`
- [ ] 5. 消费相关 `iam.lifecycle.*` Kafka Topic，收敛权限缓存
- [ ] 6. 前端菜单与按钮显隐消费 `GET /menus`（二期动态化后生效）
