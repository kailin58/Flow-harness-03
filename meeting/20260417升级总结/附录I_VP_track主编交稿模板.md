# 附录 I：11 VP track 主编一页纸交稿模板（邮件附件版）

> **性质**：`09学院（可跑顺、可闭环、可扩展）.md` **附录 I** 的轻量副本，便于**单独附件**下发主编。  
> **权威来源**：若与主文档冲突，以 [09学院（可跑顺、可闭环、可扩展）.md](./09学院（可跑顺、可闭环、可扩展）.md) 内 **「附录 I」** 及 **`§14.5` / `§14.5.6`** 为准。

---

## 用途与提交纪律

各业务 VP 主编按本文 **原样表头** 提交；**禁止删列、禁止改列名**（可增行）。平台仅校验后映射为 **`track.catalog`**（见主文档 **§14.5.3**）。`module_id` 可空由平台分配；若主编自填，须全局唯一并由 **VP04** 确认。

---

## I.1 封面行（每 VP 文件首行必填）

| 字段 | 填写说明 |
|------|----------|
| `owner_vp` | `VP01`…`VP11`（与 `AGENTS.md` 一致） |
| `track_id` | 建议 `T-VP01`…`T-VP11` 或与平台约定枚举 |
| `track_title` | 中文主轴名（可与 VP 中文名并列） |
| `chief_editor` | 主编姓名 + 角色 |
| `submitted_at` | ISO 8601 日期 |
| `director_lines_used` | 本 track 涉及的总监条线关键词列表（逗号分隔） |

---

## I.2 模块明细表（复制块；每模块一行）

| `level`（基础/进阶/专家） | `module_title` | `kind`（`common` / `role_specific`） | `director_line` | `target_role`（简述） | `module_id`（可空） | `notes`（可选） |
|---------------------------|----------------|--------------------------------------|-------------------|------------------------|---------------------|------------------|
|  |  |  |  |  |  |  |

---

## 硬校验（平台拒绝入库若违反）

- 每个 `track` **至少 6 行**（建议 common 2 + role_specific 4，允许更多行）。
- **`kind`** 两类都须至少出现 **各 ≥1** 行。
- `level` 仅允许：`基础` / `进阶` / `专家`。
- `owner_vp` 与 `track_id` 必须与封面行一致。

---

## I.3 与附录 B 的衔接（主文档附录 B）

主编完成 **I.2** 后，由 **VP04** 组织按主文档 **附录 B** 为每个 `module_id` 补全证书与能力映射（可第二迭代）；**第一迭代** 仅目录入库也可，但须在 **`§17.2` 或会议纪要** 中声明阶段范围。
