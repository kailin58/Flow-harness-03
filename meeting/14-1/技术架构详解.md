# RTK 技术架构详解

## 一、核心设计理念

### 1.1 零异步架构

RTK 采用纯同步设计，避免 async/await 带来的运行时开销：

```rust
// ❌ 禁止: async增加5-10ms启动时间
async fn run_command() -> Result<()> { ... }

// ✅ 正确: 同步执行，最小开销
fn run_command() -> Result<()> {
    let output = Command::new("git")
        .args(&["status"])
        .output()
        .context("Failed to execute git")?;
    Ok(())
}
```

**原因分析**:
- Tokio运行时初始化: ~3-5ms
- Future调度开销: ~1-2ms
- 对<10ms启动目标致命

### 1.2 退出码传播机制

```rust
// 关键模式: 正确传播底层命令退出码
pub fn run(args: &[String], verbose: u8) -> Result<i32> {
    let output = Command::new("git")
        .args(args)
        .output()
        .context("Failed to execute git")?;

    // 打印过滤后输出
    let filtered = filter_output(&String::from_utf8_lossy(&output.stdout));
    println!("{}", filtered);

    // 关键: 传播退出码而非返回Result
    if !output.status.success() {
        let code = output.status.code().unwrap_or(1);
        std::process::exit(code);
    }

    Ok(0)
}
```

**为什么重要**:
- CI/CD依赖退出码判断成功/失败
- Pre-commit钩子需要正确信号
- Git工作流要求准确传播

### 1.3 故障安全模式

```rust
// 过滤失败时的回退策略
pub fn filter_output(input: &str) -> String {
    match parse_and_filter(input) {
        Ok(filtered) => filtered,
        Err(e) => {
            // 永远不让用户得不到输出
            eprintln!("rtk: filter warning: {}", e);
            input.to_string() // 回退到原始输出
        }
    }
}
```

## 四、Token追踪实现

### 4.1 追踪模块结构

```rust
// src/core/tracking.rs
use rusqlite::{Connection, Result as SqliteResult};
use std::sync::Mutex;

pub struct Tracker {
    conn: Connection,
}

lazy_static! {
    static ref TRACKER: Mutex<Option<Tracker>> = Mutex::new(None);
}

impl Tracker {
    pub fn new() -> SqliteResult<Self> {
        let db_path = dirs::data_local_dir()
            .unwrap_or_else(|| PathBuf::from("."))
            .join("rtk")
            .join("history.db");

        // 确保目录存在
        if let Some(parent) = db_path.parent() {
            std::fs::create_dir_all(parent).ok();
        }

        let conn = Connection::open(&db_path)?;

        // 创建表
        conn.execute(
            "CREATE TABLE IF NOT EXISTS commands (
                id INTEGER PRIMARY KEY,
                timestamp TEXT NOT NULL,
                original_cmd TEXT NOT NULL,
                rtk_cmd TEXT NOT NULL,
                input_tokens INTEGER NOT NULL,
                output_tokens INTEGER NOT NULL,
                saved_tokens INTEGER NOT NULL,
                savings_pct REAL NOT NULL,
                exec_time_ms INTEGER DEFAULT 0
            )",
            [],
        )?;

        Ok(Self { conn })
    }

    pub fn track(
        &self,
        original_cmd: &str,
        rtk_cmd: &str,
        input: &str,
        output: &str,
    ) -> SqliteResult<()> {
        let input_tokens = estimate_tokens(input);
        let output_tokens = estimate_tokens(output);
        let saved_tokens = input_tokens.saturating_sub(output_tokens);
        let savings_pct = if input_tokens > 0 {
            (saved_tokens as f64 / input_tokens as f64) * 100.0
        } else {
            0.0
        };

        self.conn.execute(
            "INSERT INTO commands (
                timestamp, original_cmd, rtk_cmd,
                input_tokens, output_tokens, saved_tokens, savings_pct
            ) VALUES (?, ?, ?, ?, ?, ?, ?)",
            params![
                chrono::Utc::now().to_rfc3339(),
                original_cmd,
                rtk_cmd,
                input_tokens,
                output_tokens,
                saved_tokens,
                savings_pct
            ],
        )?;

        // 清理旧记录
        self.conn.execute(
            "DELETE FROM commands WHERE timestamp < datetime('now', '-90 days')",
            [],
        )?;

        Ok(())
    }
}

fn estimate_tokens(text: &str) -> usize {
    (text.len() as f64 / 4.0).ceil() as usize
}
```

pub fn run(cmd: &str) -> Result<()> {
    // 解析命令
    let parts = shell_split(cmd);
    if parts.is_empty() {
        std::process::exit(1);
    }

    // 查找重写规则
    let rewritten = match parts[0].as_str() {
        "git" => rewrite_git(&parts[1..]),
        "gh" => rewrite_gh(&parts[1..]),
        "cargo" => rewrite_cargo(&parts[1..]),
        "cat" | "head" | "tail" => rewrite_read(&parts),
        "rg" | "grep" => rewrite_grep(&parts),
        // ... 其他命令
        _ => None,
    };

    match rewritten {
        Some(r) => {
            println!("{}", r);
            std::process::exit(0);
        }
        None => std::process::exit(1),
    }
}

fn rewrite_git(args: &[String]) -> Option<String> {
    if args.is_empty() {
        return None;
    }

    match args[0].as_str() {
        "status" | "diff" | "log" | "add" | "commit" | "push" | "pull" 
        | "branch" | "fetch" | "stash" | "worktree" | "show" => {
            Some(format!("rtk git {}", args.join(" ")))
        }
        _ => None,
    }
}

// src/core/toml_filter.rs
use serde::Deserialize;

#[derive(Debug, Deserialize)]
pub struct TomlFilter {
    pub name: String,
    pub description: String,
    pub pattern: String,
    pub rules: Vec<FilterRule>,
}

#[derive(Debug, Deserialize)]
#[serde(tag = "type")]
pub enum FilterRule {
    #[serde(rename = "strip_lines")]
    StripLines { pattern: String },

    #[serde(rename = "summary")]
    Summary { success: String },

    #[serde(rename = "truncate")]
    Truncate { max_lines: usize },

    #[serde(rename = "dedupe")]
    Dedupe { enabled: bool },
}

pub fn find_matching_filter(cmd: &str) -> Option<TomlFilter> {
    // 从全局和项目本地加载过滤器
    let filters = load_all_filters()?;

    for filter in filters {
        let re = Regex::new(&filter.pattern).ok()?;
        if re.is_match(cmd) {
            return Some(filter);
        }
    }

    None
}

pub fn apply_filter(filter: &TomlFilter, input: &str) -> String {
    let mut result = input.to_string();

    for rule in &filter.rules {
        result = match rule {
            FilterRule::StripLines { pattern } => {
                let re = Regex::new(pattern).unwrap();
                result.lines()
                    .filter(|l| !re.is_match(l))
                    .collect::<Vec<_>>()
                    .join("\n")
            }
            FilterRule::Truncate { max_lines } => {
                result.lines()
                    .take(*max_lines)
                    .collect::<Vec<_>>()
                    .join("\n")
            }
            // ... 其他规则
        };
    }

    result
}

### 10.1 添加新命令步骤

1. **创建模块文件**: `src/cmds/<ecosystem>/<new>_cmd.rs`
2. **定义命令枚举**: 在 `main.rs` 添加 `Commands::New`

4. **添加路由**: 在 `main.rs` match分支中调用
5. **创建测试**: `#[cfg(test)] mod tests { ... }`
6. **添加固件**: `tests/fixtures/<new>_raw.txt`

        assert!(savings >= 60.0, "Expected ≥60% savings, got {:.1}%", savings);
    }
