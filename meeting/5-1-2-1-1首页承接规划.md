# 首页承接规划

## 目的

在 `meeting/5-1-2-1交易主链路块规划.md` 已明确首页是交易主链路第一段后，需要继续冻结首页内部的结构分区、各分区优先级和配置化口径，避免后续首页成为"什么都能往里塞"的膨胀场，导致交易主线失焦。

本文件主要解决：

- 首页内部按什么逻辑分区，各分区分别承接什么
- 主导航、搜索入口、活动位、客服浮层、增强入口之间的优先级如何排列
- 哪些位置冻结为稳定结构，哪些位置交给平台后台配置
- 一期首页先做到什么程度

- 交易主链路首页段的结构冻结文档

## 当前定位

首页是用户进入统一前台后的第一落点，也是交易主链路的开端。

一句话定义：

`首页的任务是让用户快速进入交易，而不是让用户在首页停留`

首页不是内容消费场，不是运营活动主场，不是增强模块展示台。所有非交易能力在首页上的存在都必须服务于一个目标：更快地进入交易主链路。

进一步统一口径：

- 首页前台承接的是稳定容器与固定坑位，不是固定活动编排
- 当前落在首页 Tab 时，频道区、商品区、活动位和增强露出位的具体内容由平台后台配置填充
- 平台后台可以调整容器内的内容、显隐和排序，但不能改写首页结构分区与优先级

## 核心优先级冻结

首页内部各能力按以下优先级排列，高优先级在布局和视觉权重上必须压过低优先级：

| 优先级 | 能力 | 说明 |
| --- | --- | --- |
| P0 | 主导航 | 底部 Tab，冻结为首页、分类/搜索、购物车、订单、我的，不可配置改动 |
| P0 | 搜索入口 | 顶部常驻，用户找货的第一路径 |
| P1 | 搜索下方模块快捷栏 | 位于搜索下方的可扩展模块栏；当前先占位天气/商城/衣橱/行程规划，后续可继续增加 |
| P1 | 商品坑位 | 首页核心内容区，直接导向商品详情，支撑交易转化 |
| P1 | 分类/频道入口 | 金刚区或频道区，支撑分类浏览进入列表 |
| P2 | 活动坑位 | 配置化运营位，增强转化但不扰乱主线 |
| P2 | 消息中心入口 | 顶部图标入口，承接通知与待办 |
| P3 | 增强入口（对话/衣橱/行程规划） | 可配置露出，服务导购与资产沉淀 |
| P3 | 客服浮层 | 全局悬浮或固定入口，不占首页内容区主位 |
| P4 | 增长与权益露出 | 推广/积分/会员等，可配置露出，最低视觉权重 |

优先级判断原则：

- P0 不可被任何配置化入口压制或替换
- P1 是首页内容区的主导力量，占据首屏核心面积
- P2 可以与 P1 共存，但面积和视觉权重不超过 P1
- P3 必须通过平台后台配置开关与位置，默认不抢占首屏核心位
- P4 只在"我的"内默认露出，首页露出必须经配置且降级展示

## 首页结构分区

### 分区总览

建议首页按纵向分区组织，从上到下依次为：

1. 顶部状态栏与搜索区
2. 搜索下方模块快捷栏
3. 频道与分类入口区
4. 商品内容区
5. 增强与运营露出区
6. 全局浮层与固定入口

### 1. 顶部状态栏与搜索区

#### 定位

首页最顶部，承载消息入口、搜索入口和基础状态。

#### 主要承接

- 搜索入口（常驻）
- 消息中心入口（图标，未读标记）
- 基础状态提示（如登录态、未支付订单待办等）

#### 不承接

- 复杂搜索结果页（搜索结果归搜索与列表段）
- 完整消息列表页（消息列表归独立页面）

#### 设计要求

- 搜索入口必须常驻可见，不能被任何浮层或活动遮挡
- 消息入口用图标 + 未读角标，不展开内容
- 状态提示以轻量方式呈现，不阻断浏览

### 2. 搜索下方模块快捷栏

#### 定位

位于搜索入口下方的首页稳定模块容器，用于承接情境信息和增强模块的快捷入口。

#### 主要承接

- `天气` 模块占位
- `商城` 入口（会员进入客户端商城，分销进入自己商城的首页）
- `衣橱` 入口
- `行程规划` 入口
- 后续新增模块的快捷入口

#### 不承接

- 完整天气服务页面
- 商城完整首页在此直接展开后的深层编排
- 衣橱或行程规划的完整功能展开
- 因模块数量增加而无限向下扩张首页布局

#### 设计要求

- 该栏固定在搜索入口下方，不与底部一级导航抢位
- 当前首批先占位 `天气`、`商城`、`衣橱`、`行程规划`
- `商城` 入口按用户身份决定落点：会员进入客户端商城，分销进入自己商城的首页
- 后续模块可持续新增，不预设固定上限
- 当前不预设固定数量上限，但当模块超出首屏可视容量时，必须保持单栏展示并通过横向滑动统一承接，不允许改成纵向无限堆叠或新增新栏位
- 平台后台只配置模块内容、顺序、显隐、身份命中规则与跳转目标，不改写该栏展示形态
- 该栏只承担快捷进入和轻量提示，不挤占商品内容区首屏核心面积

### 3. 频道与分类入口区

#### 定位

承接用户分类浏览和快速导航的需求，把用户导向商品列表。

#### 主要承接

- 分类入口（金刚区或频道卡片）
- 热门分类快捷入口
- 与分类/搜索 Tab 联动的浏览路径

#### 不承接

- 完整商品列表（归搜索与列表段）
- 分类管理能力（归平台后台）
- 具体活动内容编排（归配置化坑位）

#### 设计要求

- 分类入口必须稳定，不能因运营策略频繁调整
- 一期分类结构由后台配置内容，但入口位置和数量冻结
- 分类入口点击后进入对应的商品列表页

### 4. 商品内容区

#### 定位

首页核心面积，直接承载商品曝光，把用户导向商品详情。

#### 主要承接

- 商品推荐坑位
- 运营选品坑位
- 活动商品坑位
- 新品/热销等主题坑位

#### 不承接

- 完整商品决策信息（归商品详情页）
- 完整活动会场替代（归流量承接与活动块）
- 搜索与筛选交互（归搜索与列表段）

#### 设计要求

- 商品内容区必须占据首屏最大面积
- 坑位内容（选品、排序、曝光策略）由平台后台配置
- 坑位结构（卡片样式、区域划分）冻结为稳定结构
- 任何坑位的点击结果都必须进入交易主链路（商品详情或列表页），不允许出现"点击后离开交易场"的路径

### 5. 增强与运营露出区

#### 定位

承接增强模块、增长权益和运营活动在首页的露出位，所有位置均为配置化。

#### 主要承接

- 活动坑位（活动会场入口、限时活动等）
- 对话入口露出
- 推广中心露出
- 积分商城/会员权益露出
- 短视频导流素材位

#### 不承接

- 增强模块的完整功能（对话归智能导购块，衣橱归资产沉淀块，行程规划归场景规划块）
- 增长与权益的规则真相源
- 独立前台系统或独立内容消费场景

#### 设计要求

- 本区所有位置均为配置化坑位，由平台后台控制显隐、排序、跳转目标、命中人群、投放时间和失效回收
- 不允许任何配置化入口突破 P2 优先级限制
- 露出入口只能起"引导进入"作用，不能在首页内展开完整功能
- 配置失效时自动回退到稳定默认态，不能留空白位或异常展示
- 增强入口的默认状态：
  - 对话入口：默认露出，但位置和面积可配置
  - 推广中心：默认不露出，仅推广员资格用户经配置可露出
  - 积分商城/会员权益：默认不露出，经配置可露出快捷入口

### 6. 全局浮层与固定入口

#### 定位

不占据首页内容区面积，以浮层或固定按钮形态存在的全局入口。

#### 主要承接

- 客服浮层/按钮
- 返回顶部
- 其他全局辅助功能

#### 不承接

- 完整客服对话界面（归智能导购与服务分流块）
- 完整对话模块界面（归智能导购与服务分流块）

#### 设计要求

- 客服入口采用全局浮层或固定按钮，不占首页内容区主位
- 浮层位置不应遮挡 P0/P1 区域
- 用户可手动收起浮层，收起后在"我的"仍可找到客服入口

## 配置化口径冻结

### 冻结为稳定结构（不可配置改动）

- 底部一级导航（首页、分类/搜索、购物车、订单、我的）
- 顶部搜索入口的常驻位置
- 搜索下方模块快捷栏的容器位置
- 消息中心入口的图标位置
- 首页分区容器（频道与分类入口区、商品内容区、增强与运营露出区）的层级与主次关系
- 频道与分类入口的区域位置和数量上限
- 商品内容区作为首页核心面积的地位

### 交给平台后台配置

- 搜索下方模块快捷栏中的模块内容、顺序、显隐、身份命中规则与跳转目标
- 商品内容区的选品策略与排序
- 活动坑位的显隐、位置、内容与跳转目标
- 增强入口（对话）的首页露出开关与位置
- 增长与权益入口的首页露出开关与位置
- 消息中心的推送策略与待办弹窗策略
- 客服浮层的展示策略

### 配置化统一约束

- 所有配置化入口都不得突破优先级表中的视觉权重限制
- 配置化入口不能改写底部一级导航
- 配置化入口不能遮挡或替代搜索入口
- 配置失效时必须自动回退到稳定默认态
- 同一位置不允许同时配置两个互斥入口，由后台配置系统保证互斥

## 与其他块/段的边界

### 1. 与搜索与列表段的边界

首页负责：

- 搜索入口露出
- 分类入口露出
- 把用户导向搜索结果页和分类列表页

搜索与列表段负责：

- 搜索交互与结果展示
- 列表筛选与排序
- 商品卡信息展示

判断原则：首页只做"引导进入"，搜索与列表做"交互与结果"。

### 2. 与流量承接与活动块的边界

首页负责：

- 活动坑位的曝光容器
- 把用户导向活动会场

流量承接与活动块负责：

- 活动会场的完整页面
- 外部导流落地页
- 短视频素材转化承接

判断原则：首页只做"坑位曝光"，活动块做"页面承接"。

### 3. 与智能导购与服务分流块的边界

首页负责：

- 对话入口在首页的露出位
- 把用户导向对话模块

智能导购与服务分流块负责：

- 对话模块的完整交互
- 顾问式问答
- 从需求理解到商品/衣橱/行程规划/客服的路径分发

判断原则：首页只做"入口露出"，导购块做"交互与分流"。

### 4. 与用户资产沉淀块/场景规划块的边界

首页负责：

- 衣橱/行程规划入口的可配置露出位
- 把用户导向对应模块

资产沉淀块/场景规划块负责：

- 衣橱/行程规划的完整功能

判断原则：首页只做"入口露出"，各模块做"完整功能"。

### 5. 与增长与权益块的边界

首页负责：

- 推广/积分/会员权益的可配置露出位
- 把用户导向对应功能

增长与权益块负责：

- 推广中心、积分商城、会员权益的完整功能

判断原则：首页只做"入口露出"，增长块做"完整功能"。

## 一期范围冻结

### 一期必须做

- 顶部搜索入口常驻
- 搜索下方模块快捷栏基础版（首批占位天气/商城/衣橱/行程规划）
- 消息中心图标入口
- 频道与分类入口基础版
- 商品内容区基础版（后台配置选品，前端稳定坑位渲染）
- 客服浮层基础版
- 对话入口首页露出（可配置开关）
- 明确的"首页 → 搜索/列表 → 详情 → 购物车 → 结算"路径贯通

### 一期预留但不展开

- 更细的首页分区实验与 A/B 能力
- 首页个性化推荐策略
- 推广中心/积分商城首页露出
- 短视频素材导流位
- 更复杂的活动坑位编排能力

### 二期再做

- 首页个性化排序
- 首页内容化增强体验
- 更复杂的首页运营实验体系
- 增强模块与首页的深度融合交互

## 关键设计要求

- 首屏必须清楚表达"去哪里买、怎么找"，不能让用户在首屏迷失
- 搜索入口必须常驻可见，是首页最高优先级交互元素之一
- 搜索下方模块快捷栏应控制为固定栏位，支持后续继续加模块，但不能无限向下挤压首屏
- 任何配置化入口都不能遮挡搜索入口和底部导航
- 商品内容区必须占据首屏核心面积，活动与增强入口不能反客为主
- 所有配置化坑位的失效必须有兜底，不允许出现空白或异常展示
- 首页的职责是"分流到交易"，不是"在首页完成交易"
- 首页不承载任何模块的完整功能，只做入口露出与流量分发

## 依赖的后台能力

- 商品中心（选品与排序策略）
- 入口与投放配置后台（坑位配置、显隐控制、命中人群）
- 消息中心（通知推送与未读标记）
- 客服与工单中心（客服入口联调）
- 搜索服务（搜索入口联调）

## 当前阶段结论

- 首页优先级已冻结：P0 主导航/搜索 > P1 搜索下方模块快捷栏/商品坑位/分类入口 > P2 活动坑位/消息入口 > P3 增强入口/客服浮层 > P4 增长与权益露出
- 首页结构已分区：顶部搜索区、搜索下方模块快捷栏、频道分类区、商品内容区、增强运营露出区、全局浮层
- 配置化口径已冻结：稳定结构不可配置改动，高变内容交给平台后台配置
- 首页只做"入口露出与流量分发"，不做任何模块的完整功能
- 一期先把搜索、搜索下方模块快捷栏、分类、商品坑位、消息、客服浮层和对话入口做实
- 搜索下方模块快捷栏一期先承接天气、商城、衣橱、行程规划四个占位，后续模块继续按配置扩展
- 其中商城入口需支持按用户身份命中落点：会员进入客户端商城，分销进入自己商城的首页
- 推广、积分等增长入口在一期首页默认不露出，经配置可露出
