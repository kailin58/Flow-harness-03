## 风险管理

| 风险 | 概率 | 影响 | 缓解措施 |

| Phase A 修改引入新 bug | 低 | 高 | 全量回归测试，分批提交 |
| Phase B 技能触发误判 | 中 | 低 | 默认不激活，需明确触发词 |
| Phase C 钩子阻塞主流程 | 中 | 高 | block 策略仅用于安全关键钩子 |

| Phase E 真实执行越过政策 | 低 | 极高 | FileOperator 强制过政策检查 |
| Phase F 外部 API 引入不稳定依赖 | 中 | 高 | 仅低风险试点 + registry + 白名单 + fallback |
