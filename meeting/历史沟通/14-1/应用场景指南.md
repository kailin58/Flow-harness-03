# RTK 应用场景与实践指南

## 一、个人开发者场景

### 1.1 日常开发工作流

**场景描述**: 开发者在编写代码时频繁运行命令

**问题**:
- 大量token消耗在冗长的命令输出上
- LLM API成本快速累积
- 上下文窗口被无价值信息填满

**RTK解决方案**:
```bash
# 安装钩子 (一次性)
rtk init -g

# 之后所有命令自动过滤
git status      # 自动 → rtk git status (85%节省)
cargo test      # 自动 → rtk cargo test (95%节省)
npm run build   # 自动 → rtk npm run build (75%节省)
```

**效果对比**:
```
原始 git log -20 输出: ~2,500 tokens
RTK过滤后输出: ~250 tokens
节省: 90%
```

### 1.2 测试驱动开发 (TDD)

**场景描述**: TDD循环中频繁运行测试

**传统问题**:
```
cargo test 输出包含:
- 编译进度 (无用)
- 测试名称列表 (冗长)
- 成功测试信息 (不需要)
- 只有失败测试才重要
```

**RTK方案**:
```bash
rtk cargo test
# 只显示:
# - 失败的测试名称
# - 失败原因
# - 错误堆栈
# Token节省: 94-99%
```

### 1.3 代码审查准备

**场景描述**: 提交PR前需要查看变更

```bash
# 查看变更摘要
rtk git diff main

# 查看提交历史
rtk git log main..HEAD

# 检查文件状态
rtk git status
```

**输出精简**:
- 只显示关键变更
- 去除冗余上下文
- 保持决策所需信息

## 二、团队协作场景

### 2.1 CI/CD 集成

**场景**: 在CI流水线中使用RTK减少日志

**GitHub Actions配置**:
```yaml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Install RTK
        run: |
          curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/master/install.sh | sh
          rtk init -g
      
      - name: Run Tests
        run: rtk cargo test  # 精简输出
      
      - name: Run Lint
        run: rtk cargo clippy --all-targets
```

**效果**:
- CI日志更简洁
- 失败原因更清晰
- 日志存储成本降低

### 2.2 团队Token预算管理

**场景**: 团队共享API预算

**监控方案**:
```bash
# 每日报告
rtk gain --history --since today

# 周报告
rtk gain --history --since "1 week ago"

# 导出报告
rtk gain --export csv > team_tokens.csv
```

**团队配置**:
```toml
# ~/.config/rtk/config.toml (共享配置)
[tracking]
enabled = true
retention_days = 30
team_mode = true

[reporting]
auto_export = true
export_path = "/shared/rtk-reports/"
```

### 2.3 代码审查流程

**场景**: 使用LLM辅助代码审查

```bash
# 生成紧凑diff
rtk git diff HEAD~1 > compact_diff.txt

# 喂给LLM审查
cat compact_diff.txt | claude "Review this diff for potential issues"
```

**效果**:
- 更少的token用于传输diff
- LLM专注于实际变更
- 审查成本降低60-80%

## 三、特定语言场景

### 3.1 Rust 项目

**典型工作流**:
```bash
# 开发循环
rtk cargo check          # 快速类型检查
rtk cargo test           # 运行测试 (只看失败)
rtk cargo clippy         # Lint检查

# 发布前
rtk cargo build --release
rtk cargo doc            # 文档生成
```

**Rust专属优化**:
- 编译错误精简 (保留关键信息)
- 测试输出过滤 (只显示失败)
- 依赖树压缩 (cargo tree)

### 3.2 JavaScript/TypeScript 项目

**典型工作流**:
```bash
# 安装依赖
rtk pnpm install

# 开发
rtk vitest run            # 测试
rtk tsc --noEmit          # 类型检查
rtk eslint src/           # Lint

# 构建
rtk next build            # Next.js
rtk playwright test       # E2E测试
```

**JS/TS专属优化**:
- npm/pnpm进度条压缩
- vitest失败聚焦
- TypeScript错误精简
- Playwright报告压缩

### 3.3 Python 项目

**典型工作流**:
```bash
# 开发
rtk pytest               # 测试
rtk ruff check .         # Lint
rtk mypy .               # 类型检查

# 依赖
rtk pip install -r requirements.txt
```

**Python专属优化**:
- pytest失败聚焦
- ruff警告精简
- mypy类型错误压缩

### 3.4 Go 项目

**典型工作流**:
```bash
# 开发
rtk go test ./...        # 测试
rtk go vet ./...         # 静态分析
rtk golangci-lint run    # Lint

# 依赖

```

### 3.5 多语言项目

**场景**: 项目包含多种语言

```bash
# 根目录工作

```

**统一管理**:

## 六、高级使用场景

### 6.1 自定义过滤器

**创建项目级过滤器**:
```bash
# 创建过滤器目录
mkdir -p .rtk/filters

# 编写TOML过滤器
cat > .rtk/filters/my-tool.toml << 'EOF'
[filter]
command = "my-tool"
strategy = "error_only"

[filter.rules]
keep_patterns = ["^error", "^Error", "^FAILED"]
max_lines = 50
EOF
```

**使用自定义过滤器**:
```bash
rtk my-tool --some-args
# 自动应用 .rtk/filters/my-tool.toml
```

### 6.2 透传模式

**场景**: 需要完整输出时

```bash
# 透传执行，不过滤
rtk proxy git log -100 > full_log.txt

# 统计显示0%节省 (输入=输出)
rtk gain --history | grep proxy
```

### 6.3 调试过滤器

**场景**: 过滤器行为不符合预期

```bash
# 详细模式查看过滤过程
rtk -v cargo test

# 输出包括:
# - 原始输出行数
# - 过滤后行数
# - 应用的过滤规则
```

### 6.4 批量命令优化

**场景**: 需要运行多个命令

```bash
# 使用脚本批量运行
#!/bin/bash
commands=(
  "git status"
  "cargo test"
  "cargo clippy"
  "npm run build"
)

for cmd in "${commands[@]}"; do
  rtk $cmd
done
```

## 八、故障排除场景

### 8.1 过滤器不工作

**诊断步骤**:
```bash
# 1. 验证安装
rtk verify

# 2. 检查命令是否被支持
rtk --help | grep <command>

# 3. 使用详细模式
rtk -v <command>

# 4. 尝试透传模式对比
rtk proxy <command>  # 原始输出
rtk <command>        # 过滤输出
```

### 8.2 性能问题

**检查启动时间**:
```bash
hyperfine 'rtk --version' --warmup 3
# 应该 <10ms
```

**如果超过10ms**:
1. 检查数据库大小
2. 清理旧记录: `rtk gain --cleanup`
3. 检查配置文件复杂度

### 8.3 钩子冲突

**场景**: 与其他工具钩子冲突

```bash
# 检查钩子状态
rtk verify

# 禁用自动重写
rtk init --no-auto-rewrite

# 手动使用RTK
rtk <command>
```
