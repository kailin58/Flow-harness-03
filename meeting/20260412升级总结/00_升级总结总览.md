## 核心决策一览

### 组织架构（横向）
- Agent层级：3层 → 4层（新增 VP 副总层）
- 数字VP + 5总监 **写死**，多行业通用
- 其他7个VP **占位**，用户自建
- COMM_RULES 从"基于Agent ID"改为"基于level"

### 工作流（纵向）★ 本次更新
- 工作流从 **6层升级为9层**
- 新增 Layer 1（入口扩展）、Layer 2（权限管理）、Layer 4（协作编排）
- 权限管理**独立成层**（Layer 2），不与安全策略混用
- 详见：[05_9层工作流设计.md](./05_9层工作流设计.md)

### 模型
- 三种来源：平台内置 / BYOK / 本地模型
- 不引入重框架，使用官方SDK
- 配置走 `.flowharness/models.json` + 环境变量引用Key

### 协作
- 补充 Task State Machine（最缺的一环）
- 添加 Agent 看板（终端版先做，Web版后做）

### 长期
- 六边形架构（Ports & Adapters）
- MCP 协议先接，A2A 占位
- 平台化方向：框架 → 平台 → 生态

## 9层工作流全景

```
┌─────────────────────────────────────────────────────┐
│  L1  入口扩展层   【预留插件】网关/请求归一化         │
│  L2  权限管理层   【独立】RBAC/租户/操作授权          │ ← 新增
├─────────────────────────────────────────────────────┤
│  L3  任务编排层   分析/拆解/分配VP（原L1）            │
│  L4  协作编排层   状态机/VP协调/冲突检测              │ ← 新增
│  L5  安全策略层   准入/合规检查（原L2）               │
│  L6  执行监控层   Monitor/AutoRetry（原L3）           │
├─────────────────────────────────────────────────────┤
│  L7  Inspector层  质检/审查（原L4）                  │
│  L8  质量门禁层   通过/拦截（原L5）                   │
│  L9  反馈闭环层   复盘/优化/出口扩展（原L6）          │
└─────────────────────────────────────────────────────┘
```

**两维度说明：**
```
横向（组织层级）：CEO → VP → 总监 → 子Agent（4层）
纵向（工作流）  ：入口 → 权限 → 编排 → 协作 → 安全
                  → 执行 → 质检 → 门禁 → 闭环（9层）
两个维度正交，互不冲突
```

## 实施优先级

```
P0 基础（先做，其他依赖它）：
  ① 六边形架构骨架 + 5个端口接口定义
  ② COMM_RULES 改为基于 level（支持VP扩展）
  ③ VP层骨架 + 数字VP写死
  ④ Task State Machine（L4协作编排层核心）
  ⑤ Agent看板（终端版）
  ⑥ L1/L2 入口+权限层占位（插件链空实现）

P1 模型管理：
  ⑦ model-registry + model-router
  ⑧ 官方SDK接入 + local-adapter（Ollama）
  ⑨ cost-controller + model-monitor

P2 扩展能力：
  ⑩ L2权限管理层完整实现（RBAC + 多租户）
  ⑪ 用户自建VP Schema校验 + 模板库
  ⑫ MCP协议适配器

P3 页面（最后）：
  ⑬ Web Agent看板
  ⑭ 模型管理页面
  ⑮ VP配置页面
  ⑯ 用量/账单页面
```
