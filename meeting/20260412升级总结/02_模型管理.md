# 主题二：模型管理

---

## 三种模型来源

```
┌──────────────────────────────────────────────┐
│  平台内置模型（SaaS提供）                      │
│  Claude / GPT / Gemini                        │
│  → 用积分消耗，平台统一管理                    │
├──────────────────────────────────────────────┤
│  BYOK（Bring Your Own Key）                   │
│  用户填自己的 API Key                          │
│  → 直接扣用户自己账单，平台不收模型费           │
├──────────────────────────────────────────────┤
│  本地模型（Local）                             │
│  Ollama / LM Studio / 私有化部署               │
│  → 填本地URL，零成本，不计费                   │
└──────────────────────────────────────────────┘

优先级：本地模型 > BYOK > 平台内置
```

---

## 模型选择机制

### Agent 声明需求，不指定模型
```javascript
// 每个Agent只说自己需要什么能力等级
const AGENT_MODEL_REQUIREMENTS = {
  'supervisor':   { tier: 'premium',  capabilities: ['reasoning', 'planning'] },
  'vp_*':         { tier: 'standard', capabilities: ['coordination'] },
  'explore':      { tier: 'standard', capabilities: ['code_understanding'] },
  'plan':         { tier: 'standard', capabilities: ['architecture_design'] },
  'general':      { tier: 'standard', capabilities: ['code_generation'] },
  'inspector':    { tier: 'economy',  capabilities: ['code_review'] },  // 省钱
  'research':     { tier: 'standard', capabilities: ['web_search'], tools: ['browser'] }
}
```

### ModelRouter 自动匹配
```
CEO          → 自动选 premium tier 最优模型
Inspector    → 自动选 economy tier（质检不需要最强，省成本）
Research     → 自动选有 web_search 工具能力的模型
本地部署时    → 所有 Agent 降级到本地模型
```

## 配置方式

### 配置文件（程序可读写，为页面预留接口）
```json
// .flowharness/models.json
{
  "models": [
    {
      "id": "claude-sonnet",
      "provider": "anthropic",
      "model": "claude-sonnet-4-5",
      "key": "${ANTHROPIC_API_KEY}",
      "tier": "standard",
      "enabled": true
    },
    {
      "id": "gpt-4o",
      "provider": "openai",
      "model": "gpt-4o",
      "key": "${OPENAI_API_KEY}",
      "tier": "premium",
      "enabled": true
    },
    {
      "id": "local-llama",
      "provider": "local",
      "url": "http://localhost:11434",
      "model": "llama3.2",
      "tier": "economy",
      "enabled": true
    }
  ]
}
```

### Key 安全存储
```bash
# .env（不进 git）
ANTHROPIC_API_KEY=sk-ant-xxx
OPENAI_API_KEY=sk-xxx

# YAML/JSON 里只写引用
"key": "${ANTHROPIC_API_KEY}"
```

## 计费设计

### SaaS场景（积分体系）
```
用户充值 → 获得积分
Agent调用 → 消耗积分（不同模型不同消耗比）
平台后台：积分 ↔ token成本 ↔ 利润空间
好处：用户不感知复杂token定价，平台灵活调整
```

### 企业内部（三层分摊）
```
VP级   → 数字VP本月花了多少
Agent级 → Plan Agent / Research Agent 各自消耗
任务级  → 每个用户任务的AI成本
```

### 本地部署
```
计费模块可完全关闭
本地模型零成本，无需统计
```

## 降级链（故障自动切换）
```
claude-opus 失败
  → 自动降级 gpt-4o
    → 还失败 → 降级 claude-sonnet
      → 还失败 → 返回错误 + 告警
```

## 模块结构

```
src/adapters/
├── model-registry.js     ← 模型注册表（管理所有可用模型）
├── model-router.js       ← 路由决策（基于需求匹配模型）
├── llm-adapter.js        ← 统一调用基类（接口定义）
│   ├── mock-adapter.js   ← 当前占位（模拟返回）
│   ├── openai-adapter.js ← GPT系列 + BYOK OpenAI Key
│   ├── claude-adapter.js ← Claude系列 + BYOK Anthropic Key
│   ├── gemini-adapter.js ← Gemini系列
│   └── local-adapter.js  ← Ollama / 任意本地HTTP接口
├── cost-controller.js    ← 积分消耗 + 成本追踪 + 预算告警
└── model-monitor.js      ← 健康检测 + 延迟监控 + 错误率
```
