## 十一、当前需求总汇与实施路径（执行版）

### 11.1 需求总汇（本轮收口）

1. 会话治理采用**单一中间层**，不拆多套会话系统。
2. 会话编排采用**策略化路由**，不写死单一协作范式。
3. 会话能力需覆盖不同对话类型，并与模型能力智能匹配。
4. 会话结果必须可审计、可追溯、可回滚，不得绕过治理链。
5. 外部协作会话（A2A/MCP/外部模型）必须遵守隔离与加密通信规则。
6. 支持并行 A/B/C 测试择优，以 `evolution.experiment`（L23）沙盒验证 + `quality.gate`（L18）门禁 + `release.publish`（L22）发布收口。

### 11.2 会话中间层（Session Orchestrator）设计

统一组件职责：

- `Session Manager`：会话对象与状态机管理
- `Context Manager`：上下文窗口、摘要、压缩与过载治理
- `Policy Guard`：权限/作用域/风险/合规拦截
- `Strategy Router`：编排策略选择与切换
- `Governance Hooks`：审计、HITL、门禁、收口
- `Memory Bridge`：`S1/S2/S3/S4` 受控读写桥接

### 11.3 编排策略（可插拔）

建议支持以下策略枚举（按任务与风险动态选择）：

- `linear`：线性编排
- `manager_worker`：管理者保持控制、执行者分工
- `handoff`：会话所有权移交
- `swarm`：并行蜂群（受控启用）；规模与 `constraints` 字段命名以 [`08_蜂群牧羊犬协议.md`](../08_蜂群牧羊犬协议.md) §四、§八为准（**禁止** `max_agents`）
- `external`：外部协作（A2A/MCP）

约束：

1. 策略可切换，但必须记录切换原因与审计事件。
2. 任意策略都必须经过统一治理钩子，不得旁路。

### 11.4 对话类型与模型智能匹配

对话类型建议：

- `chat_general` / `chat_planning` / `chat_execution`
- `chat_review` / `chat_risk` / `chat_external`
- `chat_hitl` / `chat_long_running`

匹配机制：

1. 先识别对话类型，生成能力需求向量（推理强度/上下文/工具/结构化输出/多模态/延迟/成本/风险）。
2. 再按作用域、风险与合规约束过滤模型池。
3. 在候选集评分择优，并保留备用模型。
4. 运行时允许升级/降级/切备，但必须留痕并可回滚。

### 11.5 并行 A/B/C 测试择优（写死）

自动触发规则（写死）：

1. 每次会话接收到新输入（用户输入、Agent回传、外部结果、人工补充）后，系统必须先执行一次编排评估。
2. 编排评估通过后，自动选择：
   - 不触发 A/B/C：走单策略主路径；
   - 触发 A/B/C：进入并行试跑与择优流程。
3. 触发与否不得依赖人工手工开关；人工仅可在治理链内“收紧/放宽触发阈值”，不可绕过评估流程。

触发场景：

- 高复杂规划会话
- 高风险决策会话
- 外部协作会话
- 长会话高失败率场景
- 新策略首次上线前

统一评分维度：

- 目标达成度、逻辑正确性、证据完整性、风险合规性、时延、成本、稳定性、可解释性

治理约束：

1. A/B/C 先在 `L11` 隔离域执行。
2. 仅通过 `L8` 门禁的方案可进入候选。
3. 仅通过 `L10` 发布审批的方案可切主策略。
4. 必须保留回滚点与审计链。

默认流程：

`新输入到达 -> 编排评估 -> (单策略 或 A/B/C并行) -> 评分择优 -> 门禁与发布 -> 状态回写`

### 11.6 分阶段实施路径

**P0（立即）**

- 固化会话中间层与策略路由规范（本文）
- 固化对话类型与模型匹配规则（本文 + [`02_模型管理.md`](../02_模型管理.md)）
- 固化 A/B/C 择优门禁流程（本文）

**P1（近期）**

- 会话字段标准表（`session_id`、`strategy`、`risk_level`、`trace_id` 等）与隔离键对齐
- 策略路由配置化（按任务类型、风险等级、层级角色映射）

**P2（中期）**

- 策略效果看板（成功率、成本、时延、回滚率）
- 自动调优建议（仅提案，不自动生效）

### 11.7 全局适用范围

会话治理是全局能力，不仅用于人机对话，还用于：

- Agent 间协作会话
- 外部协作会话（A2A/MCP/模型）
- HITL 会话
- 执行阶段子会话（拆解、校验、回写）

### 11.8 输出外发与数据回写分层责任（写死）

输出质检成功后，允许推送至外部平台；但外发执行、治理审批与回写职责必须分层，禁止单层全包。

责任分工：

1. `L8`（质量门禁层）：判定“可外发/不可外发”与质量准出条件。
2. `L10`（版本发布层）：判定“发到哪些平台、何时发布、回滚策略”并做发布治理回写。
3. `L6`（执行监控层 + Adapter）：执行外部接口推送，回写执行状态（成功/失败/重试/渠道ID）。
4. `L9`（反馈闭环层）：汇总外发效果与失败模式，回写复盘结论并驱动策略优化。

回写口径：

- 执行态回写：`L6`
- 发布治理回写：`L10`
- 效果复盘回写：`L9`

### 11.9 外部接口异常与监控能力（写死）

外发链路必须具备最小稳定性控制能力：

1. 幂等键：防重复推送。
2. 指数退避重试：临时失败自动重试。
3. 熔断与降级：连续失败触发熔断，转待发送队列，不阻塞主流程。
4. 死信队列：超过重试上限进入人工接管。
5. 监控看板：至少监控成功率、失败率、延迟、重试次数、回滚次数。
6. 审计留痕：推送、重试、失败、回滚全过程写审计。

### 11.10 音频输出与二次完善（补充）

输出类型补充音频维度，并支持“在现有结果上继续完善”：

- 发布版音频：`MP3` / `M4A`
- 高保真归档：`WAV`
- 字幕与脚本联动：`SRT` / `VTT` / `TXT`
- 可编辑音频工程：工程文件 + 轨道说明 + 素材索引

规则：

1. 音频不满意时，允许在当前结果上继续修剪、重配音、替换片段，不要求从零生成。
2. 二次完善沿用同一会话主线（可生成 `revision_id`）。
3. 每轮完善必须记录版本差异与修改摘要。

### 11.11 输出模式治理（自由输出 / 编排输出）

本平台输出范围统一分为两类：

1. **自由输出**：快速生成与即时反馈，适用于探索、草案、灵感阶段。
2. **编排输出**：结构化生成 -> 人工修改 -> 人工确认 -> 质检收口，适用于正式交付场景。

写死规则：

1. 内容生成模块必须支持**可控编排**，不得仅提供自由输出单路径。
2. 正式对客、对外发布、跨平台分发场景默认走“编排输出”。
3. 编排输出至少包含：模板/结构约束、版本控制、人工编辑节点、确认节点、质检节点、审计节点。
4. 自由输出可直接返回，但如进入发布链，必须补走编排与质检流程。
5. 编排输出需支持“可编辑产物”与“发布产物”双轨输出，并保持一一映射关系。

### 11.12 流程片段拼接与人工确认（写死）

流程编排完成后，允许前端将已固化流程片段拼接到当前智能状态执行。

最小规则：

1. 片段来源必须是后端版本化流程库；前端不得提交“无版本片段”入链。
2. 拼接必须声明：`segment_id`、`segment_version`、`entry_state`、`exit_state`、`依赖段`。
3. 人工确认节点可作为片段强制门禁；未确认不得推进受控后续节点。
4. 片段可并行/交汇，但必须声明依赖与冲突仲裁；收口合并由后端编排层执行并写审计。
5. 片段拼接、替换、拆分均需记录差异摘要并支持回滚。

### 11.13 后端编排与前端呈现分工（写死）

为避免流程失控，编排真源固定在后端，前端仅负责可视化与受控交互。

分工规则：

1. 后端负责编排生成、节点执行、策略路由、门禁治理、审计留痕与版本发布。
2. 前端负责流程可视化、参数填写、人工确认、节点观察与受控触发。
3. 前端不得直接创建“未注册流程”或“未版本化片段”进入执行链；前端流程变更需后端校验后生效。

### 11.14 内容生产流程智能化（目标驱动）

内容生产（图像/视频/文案/报告）统一采用“目标驱动 -> 智能编排 -> 人工确认 -> 收口发布”。

最小流程：

`输入目标 -> 结构化目标对象 -> 自动生成编排流程 -> 前端呈现流程 -> 人工确认关键节点 -> 执行与质检 -> 发布与回写`

关键要求：

1. 用户可只输入“目的/目标”，系统自动生成候选流程与节点参数。
2. 高风险节点默认插入人工确认，不得自动越过。
3. 自动编排结果必须支持版本化、回放、差异比较与回滚。

### 11.15 通用输出类型标准（办公体系）

输出类型按“发布版 + 可编辑版 + 审计元数据”三轨管理。

最小支持类型：

1. 文档与报告：`DOCX` / `PDF` / `Markdown` / `HTML` / `TXT`
2. 表格与数据：`XLSX` / `CSV` / `TSV` / `JSON` / `XML`
3. 演示与汇报：`PPTX` / `PDF Slides` / 图片化幻灯片
4. 图像与设计：`PNG` / `JPG` / `SVG` / 可编辑设计源文件
5. 视频与音频：`MP4` / `WEBM` / `MP3` / `M4A` / `WAV` / `SRT` / `VTT`
6. 报表与看板：可视化快照 + 明细数据包 + 指标口径说明
7. 审计与交付包：结论、证据、待办、责任人、截止时间、trace/版本信息

规则：

1. 正式交付场景必须输出发布版与可编辑版映射关系。
2. 所有输出必须可回写到会话与任务上下文，形成可追溯链路。
3. 不满足质检门禁的输出不得进入外发发布链。
