## 一、目标能力定义

当用户发起任务（例如“帮我开一家服装店：某网站拉货，某平台开店”）时，系统必须可完成以下闭环：

1. 目标澄清与约束识别
2. 调研与市场分析
3. 品牌与选品策略生成
4. 文案与素材生产
5. 商品发布与渠道分发
6. 运营监控与效果复盘
7. 持续学习与策略优化

---

## 一点一、虚实边界原则（写死）

1. 虚拟层仅指 Agent 的呈现形态与交互空间（角色、场景、看板叙事）。
2. 业务层必须是真实业务：真实目标、真实流程、真实数据、真实结果。
3. 虚拟呈现不得替代真实业务状态，不得覆盖真实治理链与审计链。
4. 一切决策、发布、审批、回滚、经营结论以后端真实状态与审计证据为准。

---

## 一点二、1+1+1 架构原则（写死）

明确采用 `1+1+1` 结构：

1. `1` 前端服务（展示与交互层）
2. `1` 后端服务（业务编排与治理层）
3. `1` 数据库服务（独立数据层）

硬约束：

1. 前端不得直连数据库，所有读写必须通过后端API。
2. 三层必须具备故障隔离能力，单层故障不得直接拖垮全链路。
3. 三层必须具备降级策略（如只读模式、人工接管、消息延迟重试）。
4. 逻辑分层固定为 `1+1+1`，物理部署可单机（开发/演示）或分离（生产推荐）。

## 二、全链路标准流程（写死）

### 2.0 学习前置层（先学习再执行）

- 任何新任务先进入“规则学习与编排演练”，不得直接跳执行。
- 学习对象：平台规则、目标平台政策、类目规范、历史案例、失败样本。
- 学习输出：`learning_pack`（规则摘要、禁用动作、必审节点、风险清单）。
- 准入门槛：未产出 `learning_pack` 且未通过演练，不得进入发布执行层。

### 2.1 任务接入层

- 输入：业务目标、预算范围、时间窗口、平台与渠道约束
- 输出：结构化目标卡（`goal_id`、`constraints`、`success_criteria`）

### 2.2 调研分析层

- 自动收集：品类趋势、竞品价格带、爆款特征、平台规则
- 产出：调研报告（机会点、风险点、建议优先级）

### 2.3 方案编排层

- 自动生成执行方案：品牌定位、选品策略、上新节奏、投放节奏
- 输出可执行编排图：支持人工确认节点与片段拼接
- 必须先完成“沙盘演练”：按平台规则模拟一轮完整执行并产出演练报告

### 2.4 内容生产层

- 生成：商品标题、卖点、详情文案、素材脚本（图/视频/音频）
- 规则：必须过质量门禁与合规检查后进入发布队列
- 视觉识别：素材入库前必须完成“清晰度/合规/品牌一致性/风险元素”识别

### 2.5 发布执行层

- 对接目标平台：商品创建、信息填充、价格库存同步、发布动作
- 异常处理：失败重试、熔断、人工接管、回滚
- 支付相关动作（扣费、打款、退款）默认人工干预，不允许全自动放行
- 电脑与软件操作：支持在受控边界内执行浏览器、运营后台、办公软件自动化流程
- 网页识别：执行前后自动识别页面结构、关键字段与状态码，避免误点/误填/误发布

### 2.5.1 经营策略层（定价 / 售后 / 客服）

- 定价策略：按成本、竞品、毛利目标生成价格建议与调整区间
- 售后策略：退换货规则、时效承诺、补偿策略、风险升级路径
- 客服策略：FAQ 自动应答 + 人工接管分流 + 异常工单升级
- 所有策略调整必须留痕并回写经营结果（转化、退货率、满意度、ROI）

### 2.6 运营优化层

- 监控指标：曝光、点击、转化、退货率、ROI、Token 成本收益
- 自动优化：标题/素材/价格策略 A/B/C 试跑与择优
- 视觉优化：基于素材表现与视觉识别结果自动生成替换建议（主图、细节图、短视频封面）

### 2.7 经验沉淀层

- 结果回写：策略版本、适用条件、失败原因、收益提升幅度
- 能力进化：仅在隔离域实验，通过门禁后发布

## 三、角色分工（CEO -> VP -> 总监 -> 经理）

- `CEO`：下达经营目标与预算边界，审核全局 ROI 与止损策略
- `VP`：拆解域级目标并组织跨团队资源协同
- `总监`：形成可执行业务方案并把控质量/成本平衡
- `经理`：推进迭代执行、状态回写、异常清零与收口
- `子Agent/副Deep`：执行具体作业并提供证据回传

### 3.1 分工协作硬规则（写死）

1. 单 Agent 单一任务：每个 Agent 同时只承接一个原子任务，不混合承担多个主责。
2. 主责唯一：每个原子任务只允许一个主责 Agent，其余 Agent 仅作为协作或验收角色。
3. 逐级协作：任务必须按“总监 -> 经理 -> 子Agent（-> 副Deep）”下发与回传，不得跳级。
4. 全局复用优先：编排前先检索可复用能力包，命中则复用，不重复造轮子。
5. 复用留痕：所有复用能力必须记录 `skill_id/version/scope_level` 与调用证据。
