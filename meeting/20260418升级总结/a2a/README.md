# A2A 文档目录

## 目录职责

本目录用于维护 `Agent-to-Agent`（A2A）相关文档，作为 `meeting/20260418升级总结/` 下“外部 Agent 协作”专题的长期维护位置。

适用范围：

- `VP11（External）` 第三方 Agent 接入
- `Protocol Port` 下的 `A2A Adapter`
- 外部 Agent 能力发现、任务下发、流式状态、结果回收
- 与 **2M+23L** 工作流及 **S3** 审计库的治理映射（`layer_id` 与层号以 [`../00_升级总结总览.md`](../00_升级总结总览.md) 为准；核心治理链不可绕过：`security.permissions`（L6）/`security.policy`（L14）/`quality.gate`（L18）/`release.publish`（L22））
- 隔离与越权检查与总览「**四级主体隔离**」一致（平台（项目开发者）→ 企业 → 部门 → 员工（用户））；`tenant_id` 对齐 **企业** 域（详见 [`01_A2A接口契约.md`](./01_A2A接口契约.md) 文首 **隔离口径**）

---

## 维护边界

- 本目录只维护 **A2A 专项文档**，不重复改写 `AGENTS.md` 的组织规则。
- 组织层、通信规则、S3 写入约定，以仓库根目录 `AGENTS.md` 为准。
- 六边形架构与端口分层，以 `../04_长期架构.md` 为准。
- 如与 `../00_升级总结总览.md` 冲突，以总览收口后的口径为准；总览再链接回本目录。
- 会话治理总则、外部会话在编排中的位置：见 [`../会话/README.md`](../会话/README.md)；字段与 JSON 形状：见 [`../12_会话与编排接口及字段规范.md`](../12_会话与编排接口及字段规范.md)。

## 维护原则

1. 不写“仅占位、不实现”的接口。
2. 先定义最小契约，再扩展供应商适配。
3. 协议变化仅影响 `A2A Adapter`，不反向污染内核。
4. 外部 Agent 返回结果必须重新进入内部治理链，不得直入生产结论。
