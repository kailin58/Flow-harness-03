# 13_P0实施任务清单（会话与编排）

**性质**：执行清单（从规范到落地）  
**目标**：将 `10/11/12` 三份规范转为可执行任务与验收标准  
**依赖文档**：[`10_Agent基础能力与分层裁剪规范.md`](./10_Agent基础能力与分层裁剪规范.md)、[`11_会话治理规范.md`](./11_会话治理规范.md)、[`12_会话与编排接口及字段规范.md`](./12_会话与编排接口及字段规范.md)

---

## 一、P0推进原则（写死）

1. 先跑通最小链路，再扩展能力覆盖。
2. 每个任务必须有“完成定义（DoD）”和验收证据。
3. 关键链路优先：会话中间层 -> 编排策略 -> 门禁发布 -> 回写监控。
4. 任何上线动作均遵守 `L8/L10` 门禁与回滚要求。

## 三、任务分解与验收标准

### P0-1 实施基线映射表

**目标**：把规范字段和接口映射到具体模块、责任与状态。  
**最小输出**：

- 会话字段映射（`session_id`、`strategy`、`risk_level` 等）
- 状态码映射（会话状态 + 执行状态）
- 错误码映射（`E_*` 到处理动作）
- 接口映射（后端接口 -> 前端调用点）

**DoD**：

1. 映射表覆盖 `12` 文档所有“必填字段 + 最小接口”。
2. 每行包含：负责人、模块位置、实现状态、截止时间。
3. 映射表已评审通过并可作为开发输入。

### P0-2 最小链路端到端打通

**目标**：验证核心闭环可运行。  
**最小链路**：

`目标输入 -> 后端生成编排 -> 前端呈现 -> 人工确认节点 -> 质检 -> 外发 -> 回写(L6/L10/L9)`

**DoD**：

1. 链路可重复执行（至少 2 次）。
2. 每步有状态回写与审计事件。
3. 外发失败可触发重试/熔断/人工接管。
4. 有完整执行证据（日志、状态流、审计ID）。

### P0-3 会话中间层接口定版

**目标**：冻结 V1 接口，避免前后端反复改。  
**最小接口集**：

- `createSession` / `getSession` / `updateSession`
- `pauseSession` / `resumeSession` / `settleSession`
- `generatePlanByGoal` / `evaluateStrategy` / `runABC`
- `requestHumanConfirm` / `submitHumanDecision`
- `publishArtifacts` / `rollbackPublish`

**DoD**：

1. 每个接口有请求/响应样例。
2. 明确幂等规则、错误码、重试语义。
3. 对外发布 `v1` 版本号与兼容窗口。

### P0-4 默认阈值参数表

**目标**：把“原则”变成“可执行参数”。  
**最小参数**：

- ABC 触发阈值
- 会话超时阈值
- 重试次数与退避策略
- 门禁评分阈值
- 回滚触发条件

**DoD**：

1. 参数均有默认值、上下限、适用范围。
2. 参数调整有审批链与审计记录。
3. 参数表与 `11` 文档策略一致。

### P0-5 最小监控看板上线

**目标**：具备运行可观测性。  
**最小指标**：

- 会话成功率 / 失败率 / 超时率
- ABC 触发率 / 择优收益率
- 外发成功率 / 重试率 / 回滚率
- 人工接管率

**DoD**：

1. 面板可按租户、策略、会话类型筛选。
2. 告警规则可触发通知（至少 1 条链路）。
3. 指标口径与回写口径一致（`L6/L10/L9`）。

### P0-6 运行手册（Runbook）

**目标**：异常场景可按手册处理，不依赖口头经验。  
**最小章节**：

- 外发失败处理
- 死信队列处理
- 人工接管流程
- 回滚流程
- 发布冻结与恢复

**DoD**：

1. 每个异常有“触发条件 -> 处理步骤 -> 验证步骤”。
2. 覆盖至少 3 个高频故障场景。
3. 手册与当前策略版本绑定并可追溯。

- 完成 `P0-1`、`P0-3`、`P0-4`

### 里程碑 M3（P0收口）

- 完成 `P0-6`
- 产出：上线前可执行手册与回滚保障

## 六、风险提示

1. 若 `P0-2` 未跑通，不得进入大规模策略扩展。
2. 若 `P0-5` 未上线，禁止高风险自动化外发。
3. 若 `P0-6` 缺失，禁止对外声明“可稳定运行”。
