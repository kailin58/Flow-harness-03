# AI模型管理中心规划

## 目的

AI模型管理中心不只是一个后台管理页面，而是平台级 AI 能力中枢。

本文件收口以下内容：

- 三层定位
- 治理层、语料层、调用层的职责边界
- 模型消费端清单与统一口径
- 与内容创造端、用户端对话模块、客服工作台端的关系

暂不展开：

- 具体模型清单
- 具体接口契约
- 具体语料格式
- 开发排期

## 总定位

AI模型管理中心是平台级 AI 能力中枢：

- 对上提供治理与语料管理
- 对下提供统一调用接口与模型匹配能力
- 各消费端只通过统一接口获取模型能力，不直连模型、不自建语料、不自建匹配逻辑

## 模型来源分类

平台同时管理两类模型来源，治理层、语料层、调用层对两类来源统一管控：

### 1. 本地模型

指部署在平台自有基础设施上的模型实例。

- 部署形态：自有 GPU 集群 / 容器化推理服务
- 特点：数据不出域、延迟可控、运行成本相对固定
- 治理重点：算力资源调度、模型版本热更新、实例健康度、GPU 利用率、并发上限
- 适用场景：对数据安全有严格要求的场景（用户画像、会员数据相关推理）、高频低延迟场景、需要深度定制/微调的场景

### 2. 第三方模型（API 调用）

指通过 API 调用外部模型服务商提供的能力。

- 接入形态：HTTP/gRPC API 调用（OpenAI、通义、文心、MiniMax、Stability 等）
- 特点：按量计费、能力迭代快、无需自建算力
- 治理重点：密钥安全（统一 KMS）、调用限额与成本管控、供应商健康度监控、数据脱敏（敏感数据不得明文传出）、SLA 与供应商冗余
- 适用场景：通用能力（文生图/图生图/通用对话/摘要等）、快速验证新能力、弹性扩容场景

### 模型来源选择原则

- 消费端不感知模型来源，统一通过调用层接口获取能力
- 调用层根据场景、成本、延迟、数据安全要求自动路由到本地模型或第三方模型
- 同一能力标签下可以同时注册本地模型和第三方模型，互为主备或按策略分流
- 涉及用户敏感数据的推理优先路由到本地模型
- 成本敏感或弹性场景可优先路由到第三方模型

## 三层定位

### 1. 治理层（后台管理面）

负责模型的注册、状态、标签、配额与健康度管理。

通用治理：

- 模型注册与版本管理
- 能力标签（图生图/文生图/对话理解/会话摘要/辅助回复等）
- 模型来源标记（本地 / 第三方）
- 模型状态（启用/停用/灰度/降级）
- 限额与配额管理（按场景/按消费端/按时间窗口）
- 健康度监控（成功率/时延/错误率/熔断状态）
- 模型升级与灰度发布

本地模型治理：

- 算力资源登记与调度策略
- 实例副本数与并发上限
- GPU 利用率与推理队列深度监控
- 模型文件版本与热更新管理

第三方模型治理：

- 密钥与凭证统一管理（走 KMS，禁止明文）
- 供应商登记（厂商、API 版本、区域、SLA）
- 调用成本单价与费用预警
- 数据出域规则（哪些字段必须脱敏后才能传出）
- 供应商冗余与自动切换策略

### 2. 语料层（知识底座面）

负责模型所依赖的知识、语料、领域数据的管理。

- 语料接口（灌入、版本、更新、回滚）
- 领域知识管理（穿搭知识、客服知识、内容生产知识可按场景分域）
- 语料版本控制（新版本不影响旧版本在线服务）
- 后续可扩展：微调数据管理、评测数据管理

### 3. 调用层（能力消费面）

负责为各消费端提供统一的模型调用入口与匹配能力。

统一能力：

- 统一调用接口（所有消费端通过同一 API 获取模型能力，不感知模型来源）
- 模型匹配与路由（按场景、按成本、按时延、按数据安全要求路由到本地或第三方模型）
- 降级策略（主模型不可用时自动切备选模型或返回兜底结果；本地模型故障可降级到第三方，反之亦然）
- 调用日志与成本归集（按消费端、按场景、按模型、按来源分别记录调用量与费用）

本地模型调用：

- 内部服务调用（gRPC / HTTP），延迟低、数据不出域
- 并发控制与排队策略

第三方模型调用：

- 外部 API 调用，请求前自动执行数据脱敏
- 响应超时与重试策略
- 费用实时累计与超限熔断

后续可扩展：智能路由、A/B 实验、多模型编排、本地+第三方混合推理

## 消费端清单与口径

### 当前已确认的消费端

| 消费端 | 消费场景 | 消费方式 | 典型模型来源偏好 |
|---|---|---|---|
| 用户端对话模块 | 穿搭顾问、需求理解、建议生成、服务分流 | 通过统一调用接口 | 涉及用户画像偏好本地；通用对话可走第三方 |
| 客服工作台端 | 会话理解、辅助回复、摘要、分流 | 通过统一调用接口 | 涉及订单/售后敏感信息偏好本地；摘要可走第三方 |
| 内容创造端 | AI 素材生成、模板生产、内容加工 | 通过统一调用接口 | 图生图/文生图可走第三方；涉及品牌资产可走本地 |

### 消费端统一口径

- 各消费端只通过统一调用接口获取模型能力
- 各消费端不直连模型
- 各消费端不自建语料管理
- 各消费端不自建模型匹配逻辑
- 后续新增模块如需消费模型能力，也必须走统一调用接口

## 与现有文档口径的关系

### 与内容创造端的边界

- 内容创造端负责"素材生产、审核、版本与发布前准备"
- AI模型管理中心负责"模型注册、能力治理、路由策略、语料管理、统一调用接口"
- 内容创造端消费模型能力，不替代模型治理中心

### 与用户端对话模块的关系

- 对话模块消费模型能力做穿搭顾问与服务分流
- 对话模块不承担模型治理
- 对话模块不自建语料管理

### 与客服工作台端的关系

- 客服工作台消费模型能力做会话理解、辅助回复、摘要与分流
- 客服工作台不承担模型治理
- 客服工作台不自建语料管理

## 后续新增模块如何对接

后续如果有新模块需要消费 AI 能力，统一按以下流程：

1. 在 AI模型管理中心注册消费场景
2. 配置该场景的模型匹配策略与限额
3. 消费端通过统一调用接口对接
4. 调用日志与成本自动归集到该消费端

不允许新模块：

- 绕过统一接口直连模型
- 自建独立语料管理
- 自建独立模型匹配逻辑

## 当前阶段结论

- AI模型管理中心是平台级 AI 能力中枢，不是普通后台管理页面
- 模型来源包含两类：本地部署模型 + 第三方 API 调用模型
- 三层定位：治理层、语料层、调用层，对两类来源统一管控
- 消费端不感知模型来源，统一通过调用接口获取能力
- 调用层根据场景、安全、成本、延迟自动路由到合适的模型来源
- 涉及敏感数据的推理优先本地，通用能力可走第三方
- 后续新增模块也必须走统一对接流程
